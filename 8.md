## **1. lsblk** ‚Äì List Block Devices

**Command:**

```bash
lsblk
```

**Purpose:** Shows all block devices (disks, partitions, etc.) and how they‚Äôre mounted.

**Example Output:**

```
NAME    MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
xvda    202:0    0   20G  0 disk 
‚îî‚îÄxvda1 202:1    0   20G  0 part /
xvdf    202:80   0   10G  0 disk 
```

Here:

* `xvda` ‚Üí Root disk
* `xvda1` ‚Üí Root partition mounted at `/`
* `xvdf` ‚Üí New volume, not mounted yet

## **2. df -h** ‚Äì Disk Usage (Human Readable)

**Command:**

```bash
df -h
```

**Purpose:** Shows mounted filesystems, their total space, used space, and free space.

**Example Output:**

```

/dev/xvda1       20G   12G  7.5G  62% /
tmpfs           488M     0  488M   0% /dev/shm
```

## **3. Creating, Attaching & Mounting a Volume**

Let‚Äôs assume we‚Äôre working in **AWS EC2** or any cloud VM.

---

### **Step 1 ‚Äì Create the Volume**

* In AWS Console ‚Üí EC2 ‚Üí **Volumes** ‚Üí Create Volume
* Choose **Size** (e.g., 10 GB), **Availability Zone** (must match your instance‚Äôs AZ).

---

### **Step 2 ‚Äì Attach the Volume**

* In AWS Console ‚Üí Select volume ‚Üí **Attach Volume**
* Choose your instance.
* It‚Äôll show as **`/dev/sdf`** in AWS UI, but Linux may see it as `/dev/xvdf`.

---

### **Step 3 ‚Äì Check if Linux detects it**

```bash
lsblk
```

Example output:

```
NAME    MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
xvda    202:0    0   20G  0 disk 
‚îî‚îÄxvda1 202:1    0   20G  0 part /
xvdf    202:80   0   10G  0 disk 
```

`xvdf` is our new disk ‚Äî notice it has no partitions or mount points.

---

### **Step 4 ‚Äì Format the New Volume**

```bash
sudo mkfs -t ext4 /dev/xvdf
```

* `ext4` ‚Üí Common Linux filesystem.
* This erases the volume (if it had data).

---

### **Step 5 ‚Äì Create a Mount Point**

```bash
sudo mkdir /data
```

---

### **Step 6 ‚Äì Mount the Volume**

```bash
sudo mount /dev/xvdf /data
```

---

### **Step 7 ‚Äì Verify Mount**

```bash
df -h
```

Output:

```
Filesystem      Size  Used Avail Use% Mounted on
/dev/xvda1       20G   12G  7.5G  62% /
/dev/xvdf        10G   24M  9.9G   1% /data
```

---

### **Step 8 ‚Äì Make it Permanent**

Edit `/etc/fstab`:

```bash
sudo nano /etc/fstab
```

Add:

```
/dev/xvdf   /data   ext4   defaults,nofail   0   2
```

Now it mounts automatically at boot.

### üß© The 3 Main Parts of LVM

#### 1Ô∏è‚É£ **Physical Volume (PV)**

* **What it is:** Your actual hard disk or partition.
* **Example:** `/dev/sda1`, `/dev/sdb`
* **Think of it as:** A real water tank.
* **Command to create PV:**

  ```bash
  pvcreate /dev/sdb
  pvdisplay
  ```

---

#### 2Ô∏è‚É£ **Volume Group (VG)**

* **What it is:** A collection of physical volumes combined into one big storage pool.
* **Example:** `vg_data` made from `/dev/sdb` + `/dev/sdc`
* **Think of it as:** Joining two or more water tanks together to store more water.
* **Command to create VG:**

  ```bash
  vgcreate vg_data /dev/sdb /dev/sdc
  vgdisplay
  ```

---

#### 3Ô∏è‚É£ **Logical Volume (LV)**

* **What it is:** A slice of storage space from the volume group where you can store files.
* **Example:** `lv_project` from `vg_data`
* **Think of it as:** A water tap that gives you a portion of water from the big combined tank.
* **Command to create LV:**

  ```bash
  lvcreate -L 10G -n lv_project vg_data
  lvdisplay
  ```

---

### üí° Why LVM is useful:

* You can **extend** space without reinstalling.
* You can **shrink** or **resize** easily.
* Works well with **dynamic storage needs**.

---

üìå **Quick Flow to Create LVM**

```bash
# 1. Create Physical Volume
pvcreate /dev/sdb /dev/sdc

# 2. Create Volume Group
vgcreate vg_data /dev/sdb /dev/sdc

# 3. Create Logical Volume
lvcreate -L 10G -n lv_project vg_data

# 4. Format LV
mkfs.ext4 /dev/vg_data/lv_project

# 5. Mount it
mkdir /mnt/project
mount /dev/vg_data/lv_project /mnt/project
```

## 1Ô∏è‚É£ **`lsblk`** ‚Äî *List Block Devices*

* **Purpose:** Shows **all** storage devices (disks, partitions, volumes) and their hierarchy.
* **Shows even unmounted disks**.
* **Command:**

  ```bash
  lsblk
  ```
* **Example Output:**

  ```
  NAME   MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
  xvda   202:0    0   20G  0 disk
  ‚îú‚îÄxvda1 202:1    0   19G  0 part /
  ‚îî‚îÄxvda15 202:15   0    1G  0 part /boot
  xvdf   202:80   0   10G  0 disk
  ```

  * `xvdf` here is a new attached volume ‚Äî notice it has **no mount point yet**.

---

## 2Ô∏è‚É£ **`df -h`** ‚Äî *Disk Free (Human-readable)*

* **Purpose:** Shows **mounted filesystems only** with their usage.
* Doesn‚Äôt show unmounted disks.
* **Command:**

  ```bash
  df -h
  ```
* **Example Output:**

  ```
  Filesystem      Size  Used Avail Use% Mounted on
  /dev/xvda1       19G   5G   14G  27% /
  tmpfs           500M     0  500M   0% /dev/shm
  ```

  * Notice `xvdf` is not here yet because it‚Äôs not mounted.

---

## 3Ô∏è‚É£ **Mounting a Volume**

When you attach a new volume in AWS/GCP/Azure, it appears in `lsblk` but **not** in `df -h` until you mount it.

**Steps:**

1. **Check the new disk**

   ```bash
   lsblk
   ```
2. **Partition the disk** (optional but common)

   ```bash
   sudo fdisk /dev/xvdf
   ```
3. **Format it**

   ```bash
   sudo mkfs.ext4 /dev/xvdf
   ```
4. **Create a mount point**

   ```bash
   sudo mkdir /data
   ```
5. **Mount the disk**

   ```bash
   sudo mount /dev/xvdf /data
   ```
6. **Verify**

   ```bash
   df -h
   ```

   Output now shows `/dev/xvdf` mounted at `/data`.

```bash

```

## **Scenario**

```bash
lsblk

```


### **2Ô∏è‚É£ Add or resize physical volume**

If you **attached a new EBS volume**:

```bash
sudo pvcreate /dev/xvdf
sudo vgextend myvg /dev/xvdf
```

If you **resized existing EBS volume**:

```bash
lsblk
sudo pvresize /dev/xvda2
```

---

### **3Ô∏è‚É£ Extend the Logical Volume**

#### Option A ‚Äî Extend **to a specific size**

```bash
sudo lvextend -L 15G /dev/myvg/mylv
```

(Here we make it **exactly 15 GB**)

#### Option B ‚Äî Use **all free space**

```bash
sudo lvextend -l +100%FREE /dev/myvg/mylv
```

---

### **4Ô∏è‚É£ Resize the filesystem**

If **ext4**:

```bash
sudo resize2fs /dev/myvg/mylv
```

```bash

```

### **5Ô∏è‚É£ Verify**

```bash
df -h
```

You‚Äôll see the filesystem has grown.
